# Cortex Code — Complete Guide

## What Is Cortex Code?

Cortex Code is Snowflake's **AI-driven intelligent agent** integrated into the Snowflake platform, optimized for complex data engineering, analytics, machine learning, and agent-building tasks. It uses an autonomous agent framework to interact directly with your Snowflake environment, with deep understanding of Snowflake's Role-Based Access Control (RBAC), schemas, and best practices.

## Feature Deep Dive

### 1. Agentic Coding in Workspaces (Snowsight)

Cortex Code operates as a conversational coding assistant within Snowsight Workspaces:

| Capability | Description |
|---|---|
| **Code Generation** | Generate SQL queries, create files, build data pipeline logic |
| **Code Modification** | Refine SQL, find logic/syntax errors, suggest optimizations |
| **Diff View** | Preview AI-suggested changes (insertions/deletions) before applying |
| **Code Explanation** | Ask Cortex Code to explain any existing SQL |
| **Fix SQL Errors** | Click "Fix" in the results grid when a query fails |
| **@ Catalog Context** | Type `@` to search and add tables/schemas/views as inline context |
| **Quick Actions** | Highlight SQL text to access Quick Edit, Format, Add to Chat, Explain |
| **Follow-up Questions** | Continue the conversation to iterate on generated code |

### 3. Skills — Domain-Specific Knowledge

Skills inject specialized instructions into conversations. They are Markdown files stored in `.cortex/skills/` or `.claude/skills/`.

```
SKILL LOADING PRIORITY (highest to lowest)
===========================================

  [Project]  .cortex/skills/       <-- Your project-specific skills
      |
  [User]     ~/.snowflake/cortex/skills/   <-- Your personal skills
      |
  [Global]   ~/.snowflake/cortex/skills/   <-- System-wide skills
      |
  [Session]  Added temporarily     <-- One-time skills
      |
  [Remote]   Cloned from git       <-- Shared team skills
      |
  [Bundled]  Built into Cortex Code <-- Snowflake's built-in skills
```

### 6. MCP — External Tool Integration

Model Context Protocol connects Cortex Code CLI to external tools:

```
CORTEX CODE CLI
     |
     +-- MCP Protocol
          |
          +-- [stdio]  Local tools (Git, file servers)
          |
          +-- [http]   Web APIs (GitHub, Jira, Slack)
          |
          +-- [sse]    Real-time services (monitoring)

  Configuration: ~/.snowflake/cortex/mcp.json

  Namespacing:  mcp__{server-name}__{tool-name}
  Example:      mcp__github__search
```

# Cortex Code CLI — Installation Guide

## Prerequisites

Before installing, ensure you have:

- A **Snowflake account** with the `SNOWFLAKE.CORTEX_USER` database role
- **Snowflake CLI** (`snow` command) installed on your machine
- One of the supported platforms (see below)
- Network access to your Snowflake server

### Supported Platforms

```
+--------------------+--------------------+
|     Platform       |   Architecture     |
+--------------------+--------------------+
|  macOS             |  arm64, x64        |
|  Linux             |  x64, arm64        |
|  Windows (WSL)     |  x64/amd64         |
|  Windows (Native)  |  x64               |
+--------------------+--------------------+
```

---

## Installation (Windows Native)

Open **PowerShell** and run:

```powershell
irm https://ai.snowflake.com/static/cc-scripts/install.ps1 | iex
```

### What This Command Does

| Part | Explanation |
|------|-------------|
| `irm` | Short for `Invoke-RestMethod` — downloads the script from the URL |
| `https://ai.snowflake.com/static/cc-scripts/install.ps1` | The official Snowflake-hosted installation script |
| `\|` | Pipes the downloaded script to the next command |
| `iex` | Short for `Invoke-Expression` — executes the downloaded script |

### What Happens After Running

1. The script downloads the latest `cortex` executable
2. Installs it to `%LOCALAPPDATA%\cortex` (e.g., `C:\Users\YourName\AppData\Local\cortex`)
3. Adds that directory to your system `PATH` so you can run `cortex` from anywhere

---

## Launching Cortex Code CLI

After installation, you can launch it from **three places**:

### Option 1: PowerShell

```powershell
PS C:\Users\YourName> cortex
```

### Option 2: Command Prompt (cmd.exe)

```cmd
C:\Users\YourName> cortex
```

### Option 3: Run Dialog (Win+R)

1. Press `Win + R`
2. Type `cortex`
3. Press Enter

---

## First-Time Setup: Connect to Snowflake

When you run `cortex` for the first time, a setup wizard walks you through connecting to Snowflake:

```
$ cortex

Welcome to Cortex Code CLI!

? Choose a connection:
  > my-snowflake-account
    staging-account
    More options...
```

- Select an existing connection from `~/.snowflake/connections.toml`, OR
- Choose **More options** to enter your Snowflake account details manually

## Installation on Linux / macOS / WSL

For non-Windows platforms, use:

```bash
curl -LsS https://ai.snowflake.com/static/cc-scripts/install.sh | sh
```

This installs the `cortex` executable to `~/.local/bin` and updates your shell profile's `PATH`.

## Useful CLI Options

| Command | Description |
|---------|-------------|
| `cortex` | Start a new session |
| `cortex -c production` | Connect using a specific connection |
| `cortex --continue` | Resume the last session |
| `cortex --resume <session_id>` | Resume a specific session |
| `cortex --plan` | Enable planning mode (confirms each action) |
